## 2. 무엇을 해결하나요?

- **입력이 귀찮다** — 기존 일정 앱은 폼·날짜 선택·카테고리 지정이 번거롭습니다. → 자연어 한 문장으로 여러 일정과 그 사이 순서(의존성)까지 한 번에 등록합니다.
- **동기 부여가 없다** — 할 일 목록은 지루합니다. → 완료할 때마다 XP·레벨·퀘스트로 보상해 게임처럼 즐기게 합니다.
- **AI가 뭘 했는지 안 보인다** — 보통 AI는 블랙박스입니다. → 의도/프롬프트/원본 JSON/그래프/트리플을 **그대로 노출**해 신뢰를 줍니다.

## 3. 화면 구성

화면은 좌우로 나뉩니다.

```
┌───────────────────────────┬───────────────────────────┐
│  📱 왼쪽: 스마트폰 시뮬레이터  │  🔎 오른쪽: 온톨로지 라이브 뷰  │
│  - 챗봇 대화                │  - ① 지식 그래프(최상단)       │
│  - 현재 퀘스트 카드          │  - ② 의도/프롬프트             │
│  - 기능 칩 + 입력창 + 🎤    │  - ③ SDK 요청·응답 콘솔       │
│                           │  - ④ 트리플 스토어 테이블       │
└───────────────────────────┴───────────────────────────┘
```

- **왼쪽(폰)**: 실제 스마트폰처럼 보이는 화면 안에서 챗봇과 대화합니다.
- **오른쪽(온톨로지)**: 왼쪽에서 말한 내용이 AI를 거쳐 **즉시 그래프와 표로** 그려집니다. 둘은 같은 데이터를 공유해 항상 동시에 갱신됩니다.

## 4. 사용 방법 (3단계)

### ① 시작 (온보딩)
1. 앱에 들어가면 **스플래시 화면**이 잠깐 보입니다(새로고침할 때마다 다시 재생).
2. **페르소나(동반 캐릭터)** 를 고릅니다. 고른 캐릭터의 말투로 챗봇이 응대합니다.

### ② 일정 등록 — 말하거나 입력하기
- 마이크(🎤)를 누른 채 말하거나, 텍스트로 입력합니다.
- 예) *"내일 9시에 헬스 가고, 11시에 팀 미팅. 미팅 전에 보고서 마무리해야 해."*
- 결과:
  - 헬스 / 보고서 / 팀 미팅 **3개 일정 자동 추출**
  - "보고서 → 팀 미팅" **순서(의존성)** 인식
  - 팀 미팅=메인 퀘스트, 보고서=서브, 헬스=사이드로 **우선순위 분류**
  - 오른쪽 그래프에 노드/연결선이 **부드럽게 나타남**

### ③ 진행 / 조회
- *"보고서 끝!"* → 퀘스트 완료 처리, **+XP**, 페르소나가 클리어 멘트, 해당 노드 회색 처리.
- *"오늘 뭐 남았어?"* → 남은 퀘스트 요약 응답.

## 5. 주요 기능

### 5.1 페르소나(동반 캐릭터) 4종
선택한 캐릭터의 **말투·인사말**이 챗봇 응답 톤을 바꿉니다.

| 캐릭터 | 콘셉트 | 말투 예시 |
|---|---|---|
| 사만다 | 다정한 AI 동반자 | "응, 방금 일정 내가 다 정리해뒀어." |
| 아르카나 | 판타지 길드 마스터 | "용사여, 일정은 곧 그대의 퀘스트라네." |
| 무명검선 | 무협 사부 | "제자여, 하루의 수련을 게을리 말라." |
| 제이 | 전담 비서 | "일정을 추가하겠습니다." |

### 5.2 게임화(퀘스트 · XP · 레벨)
- 퀘스트 종류별 보상: **사이드 10 / 서브 30 / 메인 100 XP**
- 누적 XP로 **레벨업**, 레벨업 시 축하 메시지.
- "지금 해야 할 일"을 **현재 퀘스트 카드**로 강조.

### 5.3 일정 관리 기능 (폰 안 바텀시트)
화면 아래에서 올라오는 시트로 8가지 보기를 제공합니다.

| 기능 | 설명 |
|---|---|
| 투두 | 미완료 일정 목록, 체크로 완료 |
| 체크리스트 | 전체 일정 + 진행률 |
| 주간 | 요일별 일정 보드 |
| 월간 | 달력 + 날짜별 상세 |
| 통계 | 레벨/XP/완료율, 분포 차트 |
| 노트 | 일정이 아닌 발화 자동 저장 + 수동 메모 |
| WBS | 작업 분해 구조(계층 트리) |
| 칸반 | 할 일 / 진행 중 / 완료 3열 보드 |

### 5.4 온톨로지 라이브 뷰 (오른쪽 패널)
- **지식 그래프**: 일정·장소·분류 노드와 그 사이 관계(BEFORE, AT_LOCATION)를 시각화.
- **의도/프롬프트**: 최근 발화의 의도와 AI에 보낸 프롬프트.
- **SDK 콘솔**: AI에 보낸 **실제 요청/응답 JSON**, 모델·응답시간·토큰·LIVE/폴백 여부 표시.
- **트리플 스토어**: (주어, 서술어, 목적어) 표 형태의 지식 저장소.

### 5.5 편의 기능
- **더보기 메뉴**: 페르소나 변경 · 친구 일정 구독 · 앱 정보 · 대화 초기화.
- **친구 일정 구독**: 친구의 공개 일정을 구독하면 알림 피드로 수신(벨 아이콘 배지).
- **진행 인디케이터**: AI 통신 중에는 타이핑 버블·전송 버튼 스피너·SDK 진행 바 등 **귀여운 애니메이션**으로 대기 시간을 채움.

## 6. 동작 흐름 (내부)

```
사용자 발화(음성/텍스트)
  → 의도 분석 요청 (POST /api/extract, 서버에서만 AI 키 사용)
  → Copilot SDK가 구조화된 결과(JSON) 반환
       └ 키 없음/실패 시 → 한국어 휴리스틱 파서로 자동 폴백 (항상 응답)
  → 일정/그래프/트리플/퀘스트 갱신 + 페르소나 말투로 응답
  → 왼쪽 챗봇과 오른쪽 온톨로지 뷰가 동시에 라이브 갱신
```

- **의도 7종**: 일정 추가 / 퀘스트 완료 / 건너뛰기 / 다음 / 조회 / 취소 / 잡담.
- 일정이 아닌 잡담은 **노트로 자동 저장**되어 손실되지 않습니다.

## 7. 기술 구성

| 영역 | 사용 기술 |
|---|---|
| 프론트엔드 | **Angular 21** (Standalone · Zoneless · Signals 전용 · 신규 제어 흐름) |
| 설계 원칙 | **SOLID** — 포트/어댑터 + 의존성 주입(교체 가능한 경계 설계) |
| 서버 | Angular SSR + **Express** (`/api/extract` AI 프록시 포함) |
| AI | **Copilot SDK**(서버 전용, 구조화 출력) + 휴리스틱 폴백 |
| 음성 입력 | Web Speech API |
| 데이터 저장 | 브라우저 LocalStorage (별도 DB 없음) |
| 배포 | **Azure App Service** (Linux, Node 22) 단일 노드 서버 |

- AI 키(`COPILOT_API_KEY`)는 **서버에서만** 읽고 클라이언트에 절대 노출하지 않습니다.
