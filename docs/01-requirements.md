# 요구사항 정의서

## 1. 기능 요구사항 (FR)

### FR-1. 음성/텍스트 입력
- FR-1.1 사용자가 마이크 버튼을 누르고 말하면 Web Speech API(`SpeechRecognition`)로 텍스트 변환.
- FR-1.2 텍스트 입력창에서도 동일한 처리 경로로 입력 가능.
- FR-1.3 입력은 한국어 우선 지원, 영어 혼용 허용.

### FR-2. 자연어 → 일정 추출 (LLM)
- FR-2.1 한 발화에서 **여러 개**의 일정/할 일을 추출한다.
- FR-2.2 각 항목에 대해 다음 속성을 추론한다:
  - `title` (string, 필수)
  - `start` / `end` (ISO8601, 옵션)
  - `location` (옵션)
  - `dependsOn` (다른 항목 id, 옵션)
  - `priority` (`main` | `side`)
  - `category` (`work` / `health` / `study` / `errand` / `personal`)
- FR-2.3 모호한 시간 표현("내일", "점심 이후")은 사용자의 로컬 시간 기준으로 해석한다.

### FR-3. 온톨로지 저장
- FR-3.1 추출된 일정은 그래프 노드(`Task`)와 관계(`BEFORE`, `AT_LOCATION`, `OF_CATEGORY`)로 저장.
- FR-3.2 같은 장소를 가진 Task는 자동으로 한 `Location` 노드를 공유한다.
- FR-3.3 선후관계가 명시되지 않더라도 시간 기준으로 암시적 순서를 계산한다.

### FR-4. 퀘스트 생성 엔진
- FR-4.1 `priority=main` 인 Task는 **Main Quest**로 승격된다.
- FR-4.2 Main Quest와 `dependsOn`/시간상 직전 Task는 자동으로 **Sub Quest**가 된다.
- FR-4.3 그 외 항목은 **Side Quest**로 분류된다.
- FR-4.4 현재 시각 `now` 기준으로 "지금 해야 할 단 하나의 퀘스트"를 결정한다(`getActiveQuest`).

### FR-5. 챗봇 안내(NPC)
- FR-5.1 활성 퀘스트는 NPC 말투("모험가여, 다음 임무는…")로 출력한다.
- FR-5.2 사용자가 "끝!" / "완료" / "다음" 발화 시 해당 퀘스트를 클리어 처리.
- FR-5.3 클리어 시 XP +10(Side), +30(Sub), +100(Main) 부여.
- FR-5.4 누적 XP에 따라 레벨 상승 및 축하 메시지 출력.

### FR-6. 사용자 대시보드 (`/`)
- FR-6.1 좌측: 챗 인터페이스.
- FR-6.2 우측 상단: 캐릭터 카드(레벨, XP 바, 오늘의 칭호).
- FR-6.3 우측 하단: 퀘스트 보드(Active / Upcoming / Done) 카드 뷰.

### FR-7. 관리자 페이지 (`/admin`) — NLU 관제 콘솔
> 상세 설계는 [10-admin-page.md](10-admin-page.md).
- FR-7.1 사용자 발화가 발생할 때마다, 관리자 페이지의 6개 패널이 **자동 갱신**된다.
  (별도 새로고침 없이 같은 상태 저장소를 구독)
- FR-7.2 패널 구성: ① 발화 타임라인 ② 인텐트 분석 결과(JSON + 프롬프트) ③ 온톨로지 그래프 시각화 ④ 트리플 스토어 표 ⑤ 퀘스트 빌드 트레이스 ⑥ 정합성 경고.
- FR-7.3 최근 50개 발화의 처리 메타데이터(`UtteranceLog`)를 인메모리에 보관하고, JSON으로 내보낼 수 있어야 한다.
- FR-7.4 `Replay`, `Reset Graph`, `Seed Demo` 액션을 제공한다.
- FR-7.5 관리자 페이지는 환경 변수 `ADMIN_ENABLED=true` 일 때만 접근 가능하며, 그 외에는 404를 반환한다.

## 2. 비기능 요구사항 (NFR)

| ID | 항목 | 기준 |
|---|---|---|
| NFR-1 | 응답 시간 | 발화 종료 후 첫 응답까지 ≤ 3초 (LLM 호출 포함) |
| NFR-2 | 오프라인 시 | 음성 인식 실패 시 텍스트 입력으로 대체 안내 |
| NFR-3 | 접근성 | 모든 핵심 액션 키보드만으로 조작 가능 |
| NFR-4 | 상태 보존 | 새로고침 시 onto graph는 localStorage로 복원 |
| NFR-5 | 보안 | LLM API 키는 서버 측에서만 사용, 브라우저 노출 금지 |

## 3. 범위 외 (대회 MVP 제외)
- 다중 사용자/계정 시스템
- 모바일 네이티브 앱
- 캘린더(구글/애플) 양방향 동기화
- 음성 출력(TTS) — 시간 남으면 추가

## 4. 성공 지표 (대회 데모용)
1. 한 문장으로 ≥ 3개 일정을 발화 → 모두 정확히 추출되는 데모 성공.
2. 시간이 흐름에 따라 "다음 퀘스트"가 자동 전환되는 데모 성공.
3. 퀘스트 클리어 → 레벨업 애니메이션 시연 성공.
